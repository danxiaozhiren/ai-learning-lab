# 00 Source Map

更新时间：2026-07-07

这个文件维护长期追踪的资料源。它的作用不是收集链接，而是帮助判断“什么值得读、读完要产出什么”。

## 来源优先级

1. 官方发布、官方文档、官方工程博客。
2. 论文、技术报告、源码和 benchmark。
3. 一线工程师的复盘文章和会议演讲。
4. 媒体报道、社区讨论和二手总结。

## 单条资料登记格式

```text
日期：
标题：
来源：
链接：
所属方向：
为什么要读：
读完后的产出：
是否需要实验：
```

## 选题过滤标准

- 是否代表新能力、新架构、新产品形态或新工程实践？
- 是否能用一个小实验验证？
- 是否和自己的工作、产品设计、研发效率或知识沉淀有关？
- 是否值得写成文章，还是只需要做资料卡片？
