# 01 Learning Roadmap

这个路线图按“先建立判断框架，再做实验，再形成文章”的节奏设计。

## Phase 1: 建立 AI 技术地图

目标：知道当前 AI 技术生态分成哪些层，每层在解决什么问题。

产出：

- 一篇 `notes/` 技术地图笔记。
- 一张 AI 技术分层图。
- 一个后续 4 周学习列表。

重点问题：

- 模型能力、产品能力和工程能力分别是什么？
- Agent、RAG、fine-tuning、workflow automation 的边界在哪里？
- 哪些方向适合自己做实验，哪些只需要跟踪？

## Phase 2: Agent 工程主线

目标：理解 Agent 从 demo 走向可用系统，需要哪些工程能力。

产出：

- Agent 架构笔记。
- 一个最小可运行 agent demo。
- 一篇关于 Agent 工程化边界的文章草稿。

重点问题：

- Tool calling、planning、memory、evals 分别解决什么问题？
- 单 agent、多 agent、workflow 的取舍是什么？
- 如何评估 agent 是否真的变好了？

## Phase 3: RAG 与知识系统

目标：掌握企业知识接入和上下文工程的基本方法。

产出：

- RAG 架构笔记。
- 一个小型文档问答实验。
- 一份失败案例清单。

重点问题：

- 什么时候需要 RAG，什么时候只需要更好的 prompt/context？
- chunk、embedding、rerank、citation 如何影响质量？
- 如何做知识更新和权限边界？

## Phase 4: Evals 与质量控制

目标：把“感觉更好”变成可验证的质量判断。

产出：

- evals 方法笔记。
- 一个小评测集。
- 一个自动评分或人工评分表。

重点问题：

- 什么任务适合自动评测，什么必须人工评审？
- 如何设计 regression test？
- 如何处理 hallucination、格式错误和工具调用失败？

## Phase 5: AI Infra 与开源模型

目标：理解模型部署、推理成本、开源模型和本地运行的基本取舍。

产出：

- 推理部署笔记。
- 一个本地或 API 模型对比实验。
- 一份成本/延迟/效果对比表。

重点问题：

- API 模型、本地模型、私有化部署的取舍是什么？
- 延迟、吞吐、上下文长度、成本如何影响产品设计？
- 开源模型什么时候值得投入？

## Phase 6: 文章与个人方法论

目标：把学习成果转成能复用、能分享、能指导后续行动的文章。

产出：

- 3-5 篇文章草稿。
- 一个个人 AI 学习方法论。
- 一个年度 AI 技术观察清单。
